# Initial Concept
Я хочу создать нейро-бота, который будет раз в 2 минуты смотреть экран пользователя, слушать аудио с микрофона + динамиков и комментировать его действия

# Product Guide

## 1. Видение и цели
Создание максимально гибкого и проактивного ИИ-компаньона ("Приятель") для Windows, который видит экран, слышит звук и общается с пользователем, адаптируясь под любые технические и личные требования.

## 2. Ключевые функции
- **Визуальное восприятие (VLM):** Захват всего экрана и генерация .webm клипов для нативного мультимодального анализа контекста (Gemini 2.0+).
- **Слуховое восприятие:** Захват микрофона и системного звука (WASAPI Loopback) с визуализацией уровня через VU-метр.
- **Гибкий аудио-вход:** Выбор между STT-схемой (Whisper) и нативной мультимодальностью (прямое аудио в модель через кольцевой буфер).
- **Голосовой ответ (TTS):** Опциональный синтез речи с использованием Edge-TTS (локально) или облачных API.
- **Проактивность:** Логика «внутреннего желания», регулируемая пользователем.
- **Приватность и безопасность:** Чёрный список приложений, Offline-Only режим, маскирование чувствительных данных, журнал передачи данных, полная очистка данных (Wipe).

## 3. Интеллект и память
- **Управление контекстом:** Сжатие истории (400к+ токенов) с сохранением последних 10 запросов.
- **Гибридная память:** Долгосрочные воспоминания (RAG) + контекст сессии.
- **Эмоциональный интеллект:** Реакция на эмоции пользователя и контекст ситуации.

## 3.5 Конструктор личности (Personality Constructor)
- **Библиотека 300+ prompt блоков:** Модульная система сборки личности ИИ из готовых блоков.
- **10 категорий блоков:** Roles (роли), Styles (стили), Behaviors (поведение), Constraints (ограничения), Intelligence (интеллект), Humor (юмор), Habits (привычки), Context (контекст), Tone (тон), Special Modes (спецрежимы).
- **Drag-and-Drop интерфейс:** Визуальная сборка промпта из блоков с возможностью поиска и фильтрации.
- **Поддержка переменных:** Динамическая подстановка значений через {{variables}}.
- **Условная логика:** Блоки могут требовать или конфликтовать с другими блоками.
- **Модульное хранение:** JSON файлы в `assets/prompt_blocks/` для удобной поддержки и расширения.

## 4. Экстремальная кастомизация (UI/UX)
Система должна предоставлять пользователю полный контроль над всеми внутренними процессами.
- **Глубина настройки:** Минимум 50 значимых параметров, каждый из которых реально влияет на поведение, точность или производительность системы. Никаких настроек "для галочки".
- **Структура:** Минимум 7 тематических вкладок настроек, в каждой не менее 4-х пунктов.
- **Вкладка AI & API Settings:** Настройка провайдеров (OpenAI/Gemini/Claude/Custom), управление API ключами с маскировкой, параметры генерации (Temperature, Top-P, Max Tokens), тест соединения.
- **Вкладка Privacy & Security:** Чёрный список приложений (Blacklist), Offline-Only режим, маскирование чувствительных данных с настраиваемыми паттернами, журнал передачи данных, полная очистка данных (Wipe).
- **Вкладка Memory Settings:** Управление краткосрочной и долгосрочной памятью (RAG), настройка триггеров суммаризации, выбор моделей эмбеддингов и полная очистка истории.
- **Преднастройки (Presets):** Наличие готовых профилей (пресетов) для быстрого старта (например, "Тихий помощник", "Активный критик", "Максимальная приватность") для тех, кто не хочет настраивать всё вручную.
- **Документация внутри UI:** Каждый пункт сопровождается максимально подробным описанием: за что отвечает, на что влияет и какие побочные эффекты может вызвать изменение (нагрузка на GPU, расход токенов, задержка ответа).
- **Модульность:** Возможность полного отключения любых модулей (TTS, Vision, Memory, STT) на лету.

## 5. Технологический стек
- **Платформа:** Windows + NVIDIA GPU.
- **Модели:** Поддержка OpenAI, Gemini, Claude и локальных моделей через любые API-схемы и кастомные `base_url`.

## 6. Безопасность и фильтрация
- Интеллектуальное отсеивание шума нейросетью с возможностью настройки «чувствительности» фильтров.
