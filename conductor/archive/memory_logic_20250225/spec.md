# Specification: Memory & Context Logic

## 1. Обзор
Разработка логического ядра для управления контекстом диалога и долгосрочной памятью.

## 2. Функциональные требования
- **Контекст:** Единый метод подсчета токенов (API или Local).
- **Сжатие:** Фоновая саммаризация при достижении лимита.
- **Очередь:** Сохранение последних N сообщений без изменений.
- **RAG:** Семантический поиск по памяти (FAISS/ChromaDB) и авто-извлечение фактов.
- **Интеграция:** Связь с ConfigManager и передача данных в UI.

## 3. Критерии приемки
- Счетчик токенов работает.
- Сжатие истории не блокирует основной поток.
- RAG извлекает релевантные факты.
- Покрытие тестами 95%.
