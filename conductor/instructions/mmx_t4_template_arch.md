# Инструкция для Minimax M2.5 (MMX): Трек №4
## Цель: Реализация Template Architect (Модуль 10+ шаблонов)

Вы — Principal Software Architect. Ваша задача — разработать гибкую систему хранения и парсинга шаблонов, на которых будут строиться все гайды.

### Протокол действий:
1. **Инициализация:**
   - Прочитайте `conductor/tracks.md`. Найдите **Track 4: Template Architect**.
   - Немедленно обновите статус трека с `[ ]` на `[~]`.

2. **Анализ задачи:**
   - Перейдите в `conductor/tracks/template_architect_20250225/`.
   - Изучите `spec.md` и `plan.md`.

3. **Исполнение:**
   - Разработайте структуру файлов для шаблонов в `assets/templates/` (например, YAML или Markdown с метаданными).
   - Реализуйте парсер шаблонов в `utils/template_parser.py`.
   - Создайте базовый набор из 10+ пустых (скелетных) шаблонов согласно спецификации.
   - Убедитесь, что шаблоны поддерживают переменные (динамическую подстановку данных из конфига).

4. **Завершение:**
   - Обновите статус трека в `tracks.md` на `[x]`.
   - Выполните коммит: `git add . && git commit -m "feat(templates): Design and implement template architecture"`.
